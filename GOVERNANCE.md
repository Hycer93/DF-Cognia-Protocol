# DF-Cognia Protocol (DFCP) —— 项目治理规则

## 一、项目使命

DF-Cognia Protocol (DFCP) 是AI世界的通用认知调度与持续记忆互操作标准。本项目旨在建立一套开放、中立、可持续演进的行业协议，让任何遵循此规范的AI Agent都能被统一识别、调度和记忆。

## 二、决策机制

### 2.1 核心维护者

本项目的核心维护者为创始团队成员，拥有对协议规范、Schema定义、附录文件的最终审核权与合并权。

### 2.2 提案流程

任何对协议规范的修改建议，需通过以下流程推进：

1. **提交Issue**：在GitHub仓库提交Issue，描述修改建议的背景、动机和预期影响。
2. **公开讨论**：核心维护者和社区参与者在Issue中进行公开讨论。
3. **提交PR**：讨论达成共识后，提交Pull Request，附带完整的修改内容和更新后的版本号。
4. **审核与合并**：核心维护者审核PR，确认修改方案的合理性、向后兼容性和安全性。审核通过后合并至主分支。

### 2.3 版本管理

协议版本号遵循语义化版本规范（SemVer）：

- **主版本号**：协议架构发生重大变更，或存在不向后兼容的修改。
- **次版本号**：新增协议功能、Schema字段或附录文件，保持向后兼容。
- **修订号**：文档勘误、措辞优化、非功能性修改。

### 2.4 协议解释权

当协议规范存在歧义或争议时，由核心维护者做出最终解释。解释结论将在协议下一个小版本中正式更新。

## 三、角色定义

### 3.1 核心维护者

创始团队成员，拥有协议规范的最终审核权。当前核心维护者名单见 `AUTHORS.md`。

### 3.2 贡献者

任何提交了被合并PR的个人或组织。贡献者名单记录在 `AUTHORS.md` 中。

### 3.3 社区参与者

通过提交Issue、参与讨论、使用DFCP构建应用等方式参与本项目的所有人。

## 四、行为准则

本项目采用Contributor Covenant作为行为准则。所有参与者应遵守以下基本原则：

- 尊重不同观点与经验
- 接受建设性批评
- 以社区利益为重
- 对社区成员保持同理心

详细的贡献者行为准则见 `CODE_OF_CONDUCT.md`（待补充）。

## 五、协议演进原则

1. **向后兼容优先**：任何修改应尽可能保持与历史版本的兼容性。
2. **安全性不可妥协**：涉及安全防护、隐私保护、伦理底线的条款，不得因优化功能而被削弱或移除。
3. **开放讨论**：重大修改需公开征求社区意见，确保多方参与。
4. **文档同步**：协议正文、Schema文件、附录文件需保持版本同步，任何修改必须同时更新所有关联文件。

## 六、知识产权声明

本协议标准本身（包括规范文档、Schema定义、附录文件）通过Apache 2.0许可证发布。协议所涉及的核心技术实现可能受到相关专利保护。任何基于本协议构建兼容系统的行为，不代表自动获得相关专利的授权。
